## Configuration

All SDKs use the same core settings:

| Setting | Purpose |
| --- | --- |
| Base URL | The Orca control-plane API URL |
| API key | Bearer token for authenticated deployments |
| Tenant ID | Tenant scope for workspace resources |
| Timeout | Request or stream timeout |
| Retries | SDK retry behavior |

## Core flow

1. Create or retrieve an agent profile.
2. Submit a run.
3. Stream run events.
4. Reuse the returned session ID for follow-up work.

## Resources

SDKs expose generated clients for profiles, pools, sessions, runs, workflows, skills, MCP, storage, memory, secrets, API keys, publishing, billing, connected apps, topology, stats, and health checks.
