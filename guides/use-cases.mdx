Orca is designed for agent features that need sessions, streaming, tools, memory, storage, and clear API boundaries.

## Support triage

Create a support agent that summarizes customer messages, classifies urgency, suggests replies, and calls approved tools for customer or ticket context.

Useful Orca surfaces:

- Agent profiles for support-specific instructions.
- Skills for support policy and escalation procedure.
- MCP servers for ticketing, CRM, or docs search.
- Run streams for live progress in your support UI.

## Research assistant

Create a research agent that gathers context, compares options, writes findings, and stores reusable notes.

Useful Orca surfaces:

- Memory for persistent knowledge.
- Storage for reports and artifacts.
- Workflows for multi-step research tasks.
- SDKs for embedding the workflow in an internal tool.

## Operations automation

Create agents for repeated operational tasks such as report generation, account review, incident prep, or data cleanup.

Useful Orca surfaces:

- CLI for scheduled or scripted runs.
- Workflows for multi-step jobs.
- Usage and stats for visibility.
- Secrets and MCP servers for controlled access to external systems.

## Coding and review agents

Create coding agents that inspect repositories, produce patches, review changes, or explain code.

Useful Orca surfaces:

- Runtime-specific agent profiles.
- Sandboxed workspaces and storage.
- Run event streams for long-running tasks.
- Session continuity for follow-up instructions.
